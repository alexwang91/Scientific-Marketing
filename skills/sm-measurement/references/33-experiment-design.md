# 33 Experiment Design

## Core Position

An experiment is a decision contract. The team agrees what result will change what action before the test starts.

## Use When

Use this for A/B tests, landing pages, offers, lifecycle, creative, channel, pricing, and nudge tests.

## Method

1. Hypothesis.
2. Treatment.
3. Control.
4. Population.
5. Randomization unit.
6. Primary metric.
7. Guardrail metrics.
8. Duration and sample size.
9. Decision rule.
10. Post-test readout.

## Outputs

- Experiment brief.
- Metric plan.
- Analysis notes.
- Decision recommendation.

## Validation

The test is ready when another analyst can run it without guessing the purpose.

## Failure Modes

- No primary metric.
- Changing the hypothesis after results arrive.
- Multiple treatment changes with no interpretation plan.
- No guardrail for quality, margin, or retention.

## Language Notes

Use "win" only after the decision rule is met.
