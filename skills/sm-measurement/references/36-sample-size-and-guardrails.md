# 36 Sample Size and Guardrails

## Core Position

Small tests can teach, but they cannot carry big budget decisions without enough signal.

## Use When

Use this before experiments, holdouts, geo tests, and personalization tests.

## Method

1. Define baseline rate or value.
2. Define minimum detectable effect.
3. Estimate sample and duration.
4. Define primary metric.
5. Define guardrails: margin, churn, complaints, returns, opt-outs, brand search, customer quality.
6. Decide stop rules before launch.

## Outputs

- Sample size note.
- Guardrail table.
- Test duration.
- Stop and scale rules.

## Validation

A test is underpowered when expected traffic cannot detect a change that would matter to the business.

## Failure Modes

- Ending early.
- Reading many segments after the fact.
- Ignoring delayed outcomes.
- Optimizing conversion while harming profit.

## Language Notes

Use "directional" for underpowered learning. Do not call it conclusive.
