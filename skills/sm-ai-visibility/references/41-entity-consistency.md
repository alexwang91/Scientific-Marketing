# 41 Entity Consistency

## Core Position

AI systems need consistent facts about the brand, product, category, use case, and proof.

## Use When

Use this when AI answers describe a brand incorrectly, omit it from relevant categories, or confuse product lines.

## Method

1. List canonical brand facts.
2. Compare website, product pages, docs, press, reviews, Wikipedia or Wikidata when relevant, retailer pages, and social profiles.
3. Find contradictions.
4. Rewrite source pages with clear entities, categories, and proof.
5. Track whether answer engines update.

## Outputs

- Entity fact sheet.
- Source inconsistency list.
- Fix plan.

## Validation

A fix works when multiple answer engines repeat the correct category and proof without prompt steering.

## Failure Modes

- Using vague taglines instead of factual descriptions.
- Changing only one page.
- Ignoring retailer and review pages.

## Language Notes

Use product names, category names, and proof points consistently.
