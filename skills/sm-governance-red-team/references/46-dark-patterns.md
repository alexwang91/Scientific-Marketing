# 46 Dark Patterns

## Core Position

Behavioral design should reduce friction. It should not trap, mislead, or pressure users.

## Use When

Use this for pricing pages, checkout, subscriptions, consent, cancellation, countdowns, default choices, and scarcity.

## Method

1. Identify the decision point.
2. Check whether the user can understand the choice.
3. Check whether refusal is as easy as acceptance where appropriate.
4. Check whether urgency or scarcity is real.
5. Remove hidden costs or confusing defaults.

## Outputs

- Dark pattern risk list.
- Safer design.
- Copy edits.

## Validation

A user should be able to explain what they agreed to and how to undo it.

## Failure Modes

- Fake countdowns.
- Hidden cancellation.
- Forced continuity.
- Confusing consent.
- Preselected paid add-ons.

## Language Notes

Use "choice design" for legitimate design. Use "dark pattern" when the design hides or pressures.
