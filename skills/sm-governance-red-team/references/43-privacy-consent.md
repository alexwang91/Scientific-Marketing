# 43 Privacy Consent

## Core Position

Personalization needs clear data rights. A tactic that needs hidden data use should not ship.

## Use When

Use this for first-party data, zero-party data, sensitive data, clean rooms, CRM, retargeting, and lookalikes.

## Method

1. List data fields.
2. Classify sensitivity.
3. Name the user permission or legal basis.
4. Minimize data.
5. Give users control where needed.
6. Define retention and access.

## Outputs

- Data use table.
- Consent gap list.
- Safer alternative.

## Validation

The team should be able to explain the data use to a customer without embarrassment.

## Failure Modes

- Combining data sources without clear permission.
- Using inferred sensitive traits.
- Keeping data longer than the tactic needs.

## Language Notes

Use "data field" and "permission." Avoid "unlocking customer intelligence."
