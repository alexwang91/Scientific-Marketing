# 12 Consumer Insight JTBD

## Core Position

Consumer insight starts with the job, trigger, barrier, substitute, and language users already use. AI can organize this evidence, but it cannot turn weak evidence into truth.

## Contains

JTBD, user language, purchase triggers, barriers, substitutes, situations, decision criteria, anxiety, desired outcomes, social context, and review mining.

## Use When

Use this for audience insight, product message, review analysis, concept testing, landing page copy, and campaign planning.

## Method

1. Collect raw user language.
2. Tag each quote as trigger, pain, barrier, substitute, desired outcome, proof need, or emotion.
3. Cluster patterns without smoothing away the wording.
4. Separate facts, quotes, inference, and hypotheses.
5. Turn each insight into a message, product, offer, or experiment.

## How To Find

Use customer interviews, review sites, customer support logs, sales calls, community forums, Reddit, YouTube comments, TikTok comments, Amazon, BestBuy, app stores, and internal CRM notes.

Do not search by default. Search only when you need current public user language or category examples.

## Outputs

- Insight table.
- User quote bank.
- JTBD statement.
- Message hypotheses.
- Research gaps.

## Validation

Each insight should point to traceable user evidence or be labeled as inference. A good insight predicts a testable behavior.

## Failure Modes

- Rewriting user language into polished brand language too early.
- Mistaking a persona for a job.
- Treating loud complaints as representative without checking frequency and business impact.

## Language Notes

Use the user's words when possible. Do not replace clear quotes with marketing abstractions.
