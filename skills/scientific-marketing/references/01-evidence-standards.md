# Evidence Standards

## Core Position

Marketing teams waste money when they treat plausible stories as evidence. Separate observed facts, user language, inference, and hypotheses before recommending spend.

Evidence strength depends on the claim. Platform docs can prove how a platform works. They cannot prove a campaign will work for this business.

## Evidence Levels

- Strong: randomized experiment, holdout, geo test, credible quasi-experiment, validated causal model, validated MMM for budget scenarios, platform official docs for platform behavior.
- Medium: high-quality industry report, documented case study with method, historical internal data, repeated user evidence from a traceable source.
- Weak: vendor case study without method, single anecdote, benchmark without sample details, social thread.
- Hypothesis: AI simulation, synthetic consumer output, strategist inference, unverified trend.

## Causal Claim Rule

Use "caused" only when the evidence includes a control group, holdout, randomized test, credible quasi-experiment, or validated causal model.

Otherwise use:

- associated with
- observed alongside
- may have contributed
- is a hypothesis
- needs test

## Use When

Use this reference in every module that makes a claim about what works.

## Method

1. Tag each major claim with its evidence level.
2. Name the source type, not just the source name.
3. State the limitation.
4. Say how to verify it in the user's business.

## Outputs

Use this structure:

`Claim | Evidence level | Basis | Limitation | How to verify`

## Validation

If a claim changes budget, audience, product, promise, channel, or staffing, it needs evidence or a test plan.

## Failure Modes

- Reporting platform ROAS as incrementality.
- Treating user self-report as observed behavior.
- Treating a synthetic persona as a customer segment.
- Quoting impressive numbers without the test design.
- Using industry benchmarks to justify business-specific spend.

## Language Notes

Prefer "supports", "suggests", "is consistent with", or "needs test".

Use "proves" only for narrow claims with strong evidence.
